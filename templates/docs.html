Some helpful notes on maintaining the database

Starting up local server

With the terminal in the project root folder, activate virtual environment: source my_env/bin/activate

Start up the Flask Development Server (not for production): flask --debug run

Backing up Database from DigitalOacean droplet

Everytime new code gets commited to the Github repository, the droplet starts a new build process, wiping the SQLite database.

In order to mitigate that, we want to backup the database from the server each time before committing. This way we can import the most up-to-date version into our local development environment, to commit it again to the repository without dropping any data.

First off, navigate to the 'Console' section of the DigitalOcean web interface related to this droplet. In the terminal, open the database on the server: sqlite3 ./database.db

Then use the following to print the SQLite database to console: .dump

Copy the database content and close the connection: .exit

Restoring the database into the local environment

In the local environment, input the data into backup.sql. Make sure you keep the DROP TABLE IF EXISTS line. Actually all you really need to copy are the database entries on the bottom.

Build the up-to-date database locally. Run from terminal: python init_db.py

Commit and deploy changes to Github. That's it!
